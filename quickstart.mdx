This guide walks you through cloning the project, installing dependencies, and starting your first conversation.

## Prerequisites

- [Node.js](https://nodejs.org) 18 or later
- An [OpenAI API key](https://platform.openai.com/api-keys)

## Start the app

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/pwobus/Mandarin-tutor-v4
    cd huayu-buddy
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```

    <Note>
      You can optionally install `nodemon` globally for automatic server restarts during development:

      ```bash
      npm install -g nodemon
      ```
    </Note>
  </Step>

  <Step title="Create a .env file">
    Create a `.env` file in the project root and add your OpenAI API key:

    ```bash
    OPENAI_API_KEY=sk-your_key_here
    ```

    See [Configuration](/configuration) for the full list of available environment variables.
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```

    This command uses `concurrently` to start both servers in parallel:

    - **React UI** on `http://localhost:3000`
    - **Express API** on `http://localhost:8787`

    Open your browser to `http://localhost:3000` when both are ready.
  </Step>

  <Step title="Start a conversation">
    1. Select your **HSK level** and optionally choose a topic (for example, ordering food or travel).
    2. Click **Start Conversation**.
    3. Hold the **Hold to Talk** button and speak in Mandarin or English.
    4. The tutor responds in Mandarin with audio, Hanzi, Pinyin, and an optional English translation.
    5. Use the **HSK Review** tab to revisit vocabulary from your session.
  </Step>
</Steps>

## Conversation modes

Huayu Buddy offers two modes for spoken interaction. You can switch between them from the UI.

### Standard TTS flow

The default mode uses a request/response cycle:

1. You record audio by holding **Hold to Talk**. The recording is sent to `POST /api/stt` where OpenAI Whisper transcribes it.
2. The transcript is sent to `POST /api/chat`, which calls GPT to generate a Mandarin reply.
3. The reply text is sent to `POST /api/tts`, which returns an MP3 from OpenAI TTS.
4. The browser plays the audio and renders Hanzi + Pinyin subtitles.

Browser TTS is used as a fallback if the OpenAI TTS request fails.

<Tip>
  Standard mode is slightly higher latency than Realtime but is easier to debug and works well on slower connections.
</Tip>

### Realtime WebRTC

The Realtime panel connects directly to the OpenAI Realtime API over WebRTC for streaming, low-latency conversation:

1. The browser calls `POST /api/realtime-session` on your Express server to obtain an ephemeral session token.
2. A WebRTC peer connection is established with the OpenAI Realtime API using that token.
3. Audio streams in both directions with minimal round-trip delay.

The Realtime panel supports two models:

| Model | Notes |
|---|---|
| `gpt-4o-realtime-preview` | Default; supports custom TTS voices |
| `gpt-realtime-mini` | Lower cost; custom voices not supported |

<Warning>
  Custom TTS voices are only available when the **gpt-realtime-mini** toggle is **off**. Enabling the mini model disables voice selection.
</Warning>

## Development commands

| Command | Description |
|---|---|
| `npm run dev` | Start React UI and API server for web development |
| `npm run electron:dev` | Start Electron development mode (UI + API + desktop window) |
| `npm run electron:start` | Test a production Electron build without packaging |
| `npm run electron:build` | Package a Windows portable `.exe` into `dist/` |
